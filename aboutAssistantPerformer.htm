Assistant Performer (web version)

Summary

The aim of this project is to give a single performer maximum control over the performance of a music score displayed in a browser.
The project is written in HTML5 and Javascript. Webkit browsers (Chrome and Safari) are supported. Other browsers may work too, but this is not guaranteed.
MIDI devices are used for input and output.

The idea for a web version of the Assistant Performer arose while working on an earlier desktop application. Unlike its predecessor, the web application can use an advanced set of options and tools to control the playback of the displayed score. Building on these capabilities, users will eventually be able to play scores live, in their own time, using their own MIDI input and output devices.
A simple application would be like ordinary music-minus-one played in private, but with the live performer in control of the timing (think Mozart’s Clarinet concerto). Other applications include playing specially written scores, possibly with other players, in live concerts; controlling robot orchestras, etc.

This project is part of an effort to re-enable writing as a means of creating large-scale structures in music. Music notation collapsed in the 20th century, with catastrophic results...

Some details

The MIDI input and output devices can be selected from those attached to the user’s computer. At the moment, this is done using the Jazz plugin and JazzMIDIBridge wrapper code, so the Jazz plugin currently has to be installed before using this application. But this restriction should eventually be removed when browsers implement the standard MIDI API for browsers being developed by the W3C Audio Group (October 2012).
Note that the live performer’s control over the score need not be just via MIDI note-on and note-off messages. The use of continuous controllers is also forseen.

The score is in a hybrid SVG-MIDI format. Such scores could, in principle, be loaded from anywhere on the web — if they existed anywhere else but on this site.
The development of authoring software for such scores is a separate issue, which will doubtless be the subject of a future project here. At the moment (October 2012), I am using my Assistant Composer software, but it would be perfectly feasible for the standard music notation editors (Finale, Sibelius, capella etc.) to export in this format. Standard music notation is a superset of the required notation. The Assistant Performer simply ignores annotations such as performance instructons or tuplet symbols. The Assistant Composer already creates such scores on the basis of abstract MIDI information, so automatic transcription of Standard MIDI files is also a distinct possibility.
